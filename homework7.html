Observations

With some values of g and n the distribution is more uniform than other. When it is more uniform the value 0 is never hit.
I analyzed case A where n = 19 and g = 2, 3, 10, 17, and case B where n = 15 and g = 2, 6, 9, 12.
Case A seems to generate much more uniform distributions than case B, where the distribution hits only certain numbers and not the whole range, augmenting its predictability. If for example this system was used to generate pseudo random numbers in a range from 1 to n, with case B an attacker would have a much higher chance of guessing.
Looking more into this, it seems it has to do with multiplicative group of integers modulo n. In particular, when n is prime it is easy to find so called generators g such that the resulting distribution is uniform, we need to check if the order of g is equal to the euler totient function of n. When n is not prime, no simple formula for finding such generators is known.
